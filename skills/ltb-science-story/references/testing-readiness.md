# Test-Readiness — structuring the Science Story for the consumer-testing partner

The Science Story is not just presented to the client — the leading directions are **tested with consumers** before one is chosen for production. Build the deliverable so the testable pieces drop cleanly into the testing partner's process.

## What gets tested (three stimulus types)
- **Headlines** — the lead line of each narrative (≤10–12 words).
- **Narratives** — the 75–150 word story each cluster substantiates.
- **Claims** — individual consumer-facing claim lines.

## Make each stimulus test-ready
- **Discrete & comparable:** each headline, narrative, and claim is a standalone, self-contained unit — no cross-references, no "as mentioned above." A consumer sees one at a time.
- **Matched length/format within a set:** narratives being compared should be similar length and structure so the test measures the *idea*, not the word count.
- **Neutral labeling:** internal direction names ("The Inside-Out Story") are for the client, not the consumer — give each stimulus a neutral code (N1, N2… / H1, H2… / C1, C2…) for the test.
- **Register mix preserved:** include both science-forward and positioning-forward versions so the test tells you which register wins, not just which fact.
- **Compliance-clean:** every claim stimulus carries its disclaimer and stays within the product's regulatory class — you don't want the test invalidated by unusable language.

## The Test Plan slide (in the Stage-2 deck)
A single slide that lists, for this project:
- The directions/narratives going to test (by neutral code) and why each.
- The stimulus inventory: N headlines, N narratives, N claims.
- The primary decision KPI (persuasion / purchase intent, read as Top-2-Box) and the diagnostic guardrails (believability, uniqueness, relevance, clarity).
- Note: full methodology (designs, sample sizes, scoring, thresholds, AI synthetic pretest) lives in the Science Story Master Guide, Part 4 — the testing partner runs the field study; we hand them clean, coded, compliant stimuli.

## Hand-off package to the testing partner
A simple structured export (the dedicated format is built in the handover/test-package step): one row per stimulus — Code · Type (headline/narrative/claim) · Text · Source direction · Register · Disclaimer · KPI battery to apply. This keeps our creative judgment and the consumer's verdict cleanly separated.
